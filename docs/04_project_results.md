# 프로젝트 성과

## 성과 요약

| 구분 | 핵심 한 줄 |
| ------ | ------------ |
| **성과** | 단기 파일럿이 아닌 확장 가능한 AI 데이터 자산 확보 |
| **차별점** | 기술 도입이 아닌 '일하는 방식' 전환 |
| **의미** | AI를 실험한 조직 → AI로 일할 수 있는 조직 |

---

## CX (Customer Experience)

### 더 빨리, 더 정확하게, 맥락이 유지된 응대

기존에는 고객 문의가 채널·담당자별로 분산되어 동일한 내용을 반복 설명해야 하고, 과거 이슈 맥락이 단절되는 문제가 있었습니다.

이번 프로젝트를 통해:

- 고객 사용 현황과 이슈 히스토리가 **하나의 흐름으로 연결**
- 이전 상황을 이미 이해한 상태에서 **일관된 응대** 가능
- 사용량 초과, 반복 장애 등 **리스크 사전 감지** 기반 마련

| 기존 | 개선 후 |
| ------ | --------- |
| 담당자별 응대 편차 | 이력·맥락 기반 일관된 응대 |
| 문제 발생 후 대응 | 사전 감지·선제 대응 가능 |
| 고객 현황 파악 어려움 | 사용량·이슈 상태 즉시 확인 |
| 재계약 근거 부족 | 데이터 기반 제안 가능 |

---

## DX (Developer/Employee Experience)

### 찾는 시간은 줄고, 판단과 해결에 집중

구성원들은 더 이상 여러 시스템을 오가며 데이터를 취합하거나, 특정 인력의 기억과 경험에 의존하지 않아도 됩니다.

**자연어 질문만으로:**

- 고객 상태, 사용량 추이, 과거 이슈 맥락을 즉시 확인
- 정보 탐색과 정리 시간 크게 감소
- 신규 인력도 과거 이슈 흐름을 빠르게 파악

| 기존 | 개선 후 |
| ------ | --------- |
| SQL·엑셀 의존 | 자연어 질의로 즉시 조회 |
| 개인 경험 의존 | 공통 데이터 자산 활용 |
| 신규 인력 온보딩 부담 | 과거 이슈·히스토리 즉시 파악 |
| 반복 보고·정리 업무 | 자동화 및 AI 요약 |

## DT (Digital Transformation)

### 데이터가 쌓이는 조직 → AI로 판단하는 조직

이번 프로젝트를 통해 내부에는 단순한 데이터 정리가 아닌, **AI 활용을 전제로 한 데이터 구조와 기술 스택**이 축적되었습니다.

### 축적된 자산

| 영역 | 내용 |
| ------ | ------ |
| 데이터 전략 | AI 활용 전제 데이터 구조 내재화 |
| 기술 자산 | 에이전트·레이크하우스·워크플로우 축적 |
| 보안/거버넌스 | 가명정보·접근권한·로그 기준 정립 |
| 협업 문화 | 기획–개발–운영 동일 언어로 협업 |
| 확장성 | 메일·위키·OKR 등 즉시 확장 가능 |

## 제언

### 1. 서비스 적용 가능성 검증

다음 단계에서는 **고객에게 제공 가능한 AI 기능을 정의하고 검증**하는 것이 필요합니다.

- 검색, 요약, 분류, 대응 보조 등 고객 체감 기능 중심으로 테스트

### 2. 툴 중심 '실행형 에이전트'로 전환

단순 질의·요약 중심이 아닌, **실제 데이터를 조회하고 판단을 실행**할 수 있는 Tool 기반 에이전트가 실무 활용에 더 적합함을 확인했습니다.

### 3. 벡터화 vs 데이터 정제 전략적 선택

모든 데이터를 벡터화하는 방식이 아닌:

- **정형 데이터 정제·집계**가 효과적인 영역
- **벡터 기반 검색**이 적합한 영역

을 구분하여 검증하는 것이 성능, 비용, 유지보수 관점에서 합리적입니다.

## 활용 예시

에이전트에게 할 수 있는 질문:

```text
- 2025년 문의 접수 경로 (전화, 메일, 구글폼) 통계
- 플랫폼별 이슈 발생 빈도
- 이슈 구분(버그, 오류, 장애)에 따른 집계
- 이슈가 많았던 고객사
- 한 해 동안 가장 이슈가 많았던 키워드
```

> 이러한 질문들을 통해 **다음 서비스 업그레이드 시 어떤 부분을 우선 개선할지, 어떤 방식으로 개선해 나갈지** 인사이트를 얻을 수 있습니다.
