# 프로젝트 개요

## 추진 배경

전사에 분산된 핵심 데이터를 AI 기반 업무자동화 체계로 전환하고, 구성원과 고객이 데이터를 보다 쉽고 직관적으로 활용할 수 있는 **"AI 동료형 에이전트"를 만드는 것을 목표**로 프로젝트를 수행했습니다.

단순한 데이터 분석이나 PoC 수준의 실험이 아니라, **실제 업무 맥락에서 반복적으로 발생하는 질문·판단·보고 업무를 AI가 대신 수행할 수 있는 구조를 검증**하는 데 초점을 두었습니다.

---

## 프로젝트 목적

### 해결하고자 한 문제

| 구분 | 기존 문제점 |
| ------ | ------------- |
| 데이터 단절 | 팀 간 데이터가 분리되어 이슈 맥락이 유실됨 |
| 추적 어려움 | 고객 이슈의 실시간 추적 및 SLA 준수 확인 불가 |
| 분산 관리 | 메일, 전화, 폼 등 다채널 이슈가 누락됨 |
| 비효율 | 반복적인 조회/보고 업무에 시간 소모 |

### 목표

1. **중앙집중 관리**: 모든 고객 이슈를 단일 시스템(GitHub Issues)에서 추적
2. **정형+비정형 데이터**: AI가 활용 가능한 형태로 데이터 구조화
3. **업무 효율화**: 자연어 질의로 즉시 분석 가능한 에이전트 구현

## 프로젝트 범위

### 데이터 수집 채널

```text
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│    메일     │    │   구글폼    │    │    전화     │
└──────┬──────┘    └──────┬──────┘    └──────┬──────┘
       │                  │                  │
       ▼                  ▼                  ▼
┌─────────────────────────────────────────────────┐
│             GitHub Issues (통합 관리)            │
└─────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────┐
│          Data Platform (Bronze/Silver/Gold)      │
└─────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────┐
│              AI Agent (자연어 분석)              │
└─────────────────────────────────────────────────┘
```

### 핵심 원칙

> **모든 이슈는 최종적으로 GitHub Issue로 수렴한다.**

1. 메일 → 공용계정 전달 → GitHub Issue 자동 생성
2. 구글폼 → 응답 스프레드시트 → GitHub Issue 자동 생성
3. 전화 → 내부 기록 → GitHub Issue 수동 등록

## 기대 효과

### 부서별 활용 시나리오

| 구분 | 영업팀 | 개발/서비스팀 | 마케팅팀 |
| ------ | -------- | --------------- | ---------- |
| **무엇을** | 고객사별 이슈 건수/유형 집계, SLA 준수 여부, 반복 요청 고객 확인 | 이슈 발생 패턴 기록, 재발 이슈 재활용, R&D 과제 도출 | 고객 불편·요청 키워드 수집, 유사 고객군 관심사 도출 |
| **언제** | 월간 리포트, 계약 갱신 전, 고객 미팅 전 | 월별 점검, 장애 발생 시, 신규 기능 검토 시 | 캠페인 기획 전, 콘텐츠 주제 선정 시 |
| **왜** | 재계약·업셀링 근거, 고객 신뢰 강화 | 동일 이슈 신속 처리, 서비스 품질 향상 | 고객 니즈 기반 마케팅 메시지 |

### 질문 예시

- "최근 3개월간 이슈 건수가 가장 많은 고객사는 어디인가?"
- "지난 분기에 재발한 장애 유형은 무엇이며, 해결 평균 시간은?"
- "가장 많이 언급된 개선 요청 키워드는 무엇인가?"

## 향후 발전 가능성

```text
Phase 1                    Phase 2                    Phase 3
고객 이슈관리 도입    →    전사 확장 (SRE/솔루션)    →    고객 셀프서비스
                                                         
- GitHub 기반 통합          - 부서별 에이전트            - 고객 포털 AI
- 자연어 분석               - 추가 상품 판매 연계        - FAQ 자동 응답
- KPI 대시보드              - MA 계약 기회 발굴          - 문제 자가 해결
```

이를 통해:

- **신규 매출 기회** 창출 (추가 상품 판매, MA 계약)
- **효율성 향상** (중복 이슈 해결 시간 절감)
- **고객 만족** (셀프서비스 환경 구축)
